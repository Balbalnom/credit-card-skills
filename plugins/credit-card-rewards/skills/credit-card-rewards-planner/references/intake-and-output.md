# Intake And Output

## Intake Checklist

Ask only for the missing details needed for the user's goal. Keep the question list short when the user asks for a quick comparison.

Core inputs:

- Country and state, if relevant.
- Credit score band, not exact score, if the user is comfortable sharing.
- Current cards, approval dates, annual fees, and whether each card is personal or business.
- New personal cards opened in the last 24 months.
- Monthly spend by category: dining, groceries, gas/EV, airfare, hotels, transit/rideshare, online shopping, rent, utilities, drugstores, office supply, and other.
- Travel goals: cash back, flexible points, airline miles, hotel points, lounge access, free checked bags, elite status help, companion pass, or trip target.
- Home airport and preferred airlines/hotels.
- Annual-fee tolerance and willingness to track credits.
- Ability to meet minimum spend without overspending.
- Upcoming large expenses and dates.
- Need for 0% APR or balance transfer, amount, target payoff date, and whether purchases or transfers matter.
- Business-card eligibility and comfort applying for business cards.

## Valuation Defaults

State valuations as assumptions, not facts. If the user gives their own valuations, use those.

Default conservative values:

- Cash back: face value.
- Fixed-value bank points: redemption value shown by issuer.
- Transferable points: use a range and explain that value depends on redemption.
- Airline and hotel points: use a range unless the user has a specific redemption.
- Credits: value at face value only when the user's existing behavior naturally uses them; otherwise discount heavily or list separately.

## Comparison Table Columns

Use only columns relevant to the request:

- Card.
- Issuer.
- Public offer and source.
- Welcome bonus.
- Minimum spend and deadline.
- Annual fee.
- Net first-year value estimate.
- Ongoing earn fit.
- Key credits/perks.
- 0% APR details.
- Transfer partners or travel ecosystem.
- Important restrictions.
- Why it fits.
- Why it may not fit.
- Last checked date.

## Response Shape

For a card search:

1. State the user's goal and assumptions.
2. Present top picks in a table.
3. Explain the top 1-3 choices in plain language.
4. Include a "do not apply yet" section when timing, eligibility, or 5/24 makes a card risky.
5. End with exact next actions.

For a journey plan:

1. Start with the user's current card inventory and 5/24 count.
2. List application order by month.
3. Explain why each card is placed there.
4. Include minimum-spend deadlines and benefit reminders.
5. Add fallback options if denied or if an offer changes.
